## Week 01 – Java Object Model & Values vs Entities

- **Value vs entity**: when objects should be immutable value types vs identity-carrying entities.
- **Immutability pitfalls**: collections exposed directly, partially immutable types, builder leaks.
- **Contracts**: equals/hashCode/toString and how they interact with collections and logging.
- **Modern Java**: records and sealed types as better tools for value-like and algebraic hierarchies.
