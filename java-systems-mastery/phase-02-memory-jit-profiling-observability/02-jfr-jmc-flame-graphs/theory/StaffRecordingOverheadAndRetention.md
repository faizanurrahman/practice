## Staff: recording overhead, retention, and org policy

- **Event settings** trade detail vs overhead; production recordings should use **curated** templates.
- **Retention:** where `.jfr` files land, PII in object samples, and access control for performance data.
- **Correlation:** stitch JFR timelines with **metrics** (Saturation, GC) and **traces** for cross-service regressions.

**Staff decision:** default observability stack (JFR on demand vs continuous profiling agents) with explicit cost and risk.
